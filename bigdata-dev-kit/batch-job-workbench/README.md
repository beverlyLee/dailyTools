# 批处理作业工作台 (Batch Job Workbench)

## 功能概述

批处理作业工作台是大数据处理开发套件的核心子项目之一，专门用于 Spark/Flink 批处理作业的开发、提交和监控。

## 主要功能

### 1. SQL 脚本编辑
- 集成 Monaco Editor，提供专业的 SQL 编辑体验
- 语法高亮、自动补全、代码格式化
- 支持 Spark SQL 和 Flink SQL 语法
- 实时语法校验

### 2. Jar 包上传
- 支持拖拽上传和文件选择上传
- 支持最大 500MB 的 Jar 文件
- 上传文件自动保存到指定目录

### 3. 作业提交配置
- 作业名称、主类名、程序参数配置
- 计算引擎选择（Spark / Flink）
- 作业类型选择（批处理 / 流处理）

### 4. Job DAG 图可视化
- 可视化展示作业执行流程图
- 节点状态标识（等待中/运行中/已完成/失败）
- 图例说明，直观了解各节点状态

### 5. Executor 监控
- 实时监控 Executor 运行状态
- CPU/内存使用率展示
- 任务完成统计
- 状态实时更新（RUNNING / BUSY / IDLE）

### 6. 日志管理
- 分级别日志查看（INFO / WARN / ERROR / DEBUG）
- 按 Executor 过滤日志
- 日志实时刷新
- 日志时间戳显示

## API 接口

### 作业管理
- `GET /api/jobs` - 获取所有作业列表
- `POST /api/jobs` - 创建新作业
- `GET /api/jobs/{id}` - 获取作业详情
- `PUT /api/jobs/{id}/status` - 更新作业状态
- `PUT /api/jobs/{id}/progress` - 更新作业进度
- `PUT /api/jobs/{id}/dag` - 更新作业 DAG 图
- `DELETE /api/jobs/{id}` - 删除作业

### Jar 上传
- `POST /api/jobs/upload-jar` - 上传 Jar 文件

### Executor 管理
- `GET /api/executors/job/{jobId}` - 获取作业的 Executor 列表

### 日志管理
- `GET /api/logs/job/{jobId}` - 获取作业日志
- `GET /api/logs/job/{jobId}/level/{level}` - 按级别获取日志
- `GET /api/logs/job/{jobId}/executor/{executorId}` - 按 Executor 获取日志

## 使用流程

1. **新建作业**
   - 点击"新建作业"按钮
   - 填写作业名称、选择作业类型和计算引擎
   - 确认创建

2. **编写 SQL 脚本**
   - 在 Monaco Editor 中编写 Spark/Flink SQL
   - 点击"格式化"进行代码美化
   - 点击"校验"检查语法正确性
   - 点击"执行"运行 SQL

3. **上传 Jar 包（可选）**
   - 切换到 "Jar 上传" 标签页
   - 拖拽 Jar 文件到上传区域，或点击选择文件
   - 填写作业提交配置（主类名、参数等）
   - 点击"提交作业"

4. **查看 DAG 图**
   - 选择已创建的作业
   - 切换到 "DAG 图" 标签页
   - 查看作业执行流程和节点状态

5. **监控 Executor**
   - 切换到 "监控" 标签页
   - 查看 Executor 统计数据
   - 查看各 Executor 详细状态

6. **查看日志**
   - 切换到 "日志" 标签页
   - 按级别或 Executor 过滤日志
   - 点击"刷新"获取最新日志

## 注意事项

1. 确保后端服务已启动（http://localhost:8080）
2. 首次使用需要安装前端依赖：`npm install`
3. Jar 文件上传前确保后端上传目录有足够权限
4. 大型作业建议在本地测试后再提交到生产环境
