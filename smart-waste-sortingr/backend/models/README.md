# 模型文件说明

## 模型格式
本项目支持以下模型格式：
- ONNX 格式 (.onnx) - 推荐使用
- PyTorch 格式 (.pt, .pth)

## 模型要求
1. 输入尺寸: 224 x 224 像素
2. 输入通道: 3 (RGB)
3. 输入归一化: 像素值除以 255，范围 [0, 1]
4. 输出类别数: 29 类（详见下方类别映射）

## 如何添加自己的模型

### 方法 1: 使用 ONNX 模型（推荐）
1. 将训练好的 ONNX 模型命名为 `waste_classifier.onnx`
2. 将模型文件放置在本目录 (`backend/models/`)
3. 重启后端服务，模型将自动加载

### 方法 2: 使用 PyTorch 模型
1. 修改 `app/services/model_service.py` 文件
2. 在 `_load_model` 方法中添加 PyTorch 模型加载逻辑
3. 示例代码：
```python
import torch
from torchvision import models

def _load_model(self):
    # ... 现有代码 ...
    # 添加 PyTorch 模型支持
    pt_path = MODELS_DIR / "waste_classifier.pt"
    if pt_path.exists():
        self._model = torch.load(pt_path, map_location=torch.device('cpu'))
        self._model.eval()
        self._is_loaded = True
```

## 模型训练建议
1. 数据集建议: 使用公开的垃圾分类数据集，如:
   - TrashNet 数据集
   - 中国生活垃圾图像数据集
   
2. 模型架构建议:
   - MobileNetV2 / MobileNetV3 (移动端友好)
   - EfficientNet (高精度)
   - ResNet-18 / ResNet-50

3. 训练参数:
   - 图像尺寸: 224 x 224
   - 批次大小: 32-64
   - 学习率: 0.001
   - 优化器: Adam
   - 损失函数: CrossEntropyLoss

## 回退机制
当模型文件不存在时，系统会使用基于图像颜色的回退分类器。该分类器可以提供基本的分类功能，但精度较低。建议在生产环境中部署训练好的模型。
